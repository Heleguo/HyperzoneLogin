首先，统一将需要验证的玩家置入limbo服务器
# 概述
将所有的验证形式统一为一种验证类型：输入一定参数，等待成功验证  
设计成Module，插件通过Module向主程序注册来进行联机验证和验证流程控制  
其中Yggdrasil只是传入参数相对特别的Module，会丢出验证事件来传递数据，允许Module提前开始验证。  
成功验证回传信息：  
- UUID (可选，无实际用途)
- name (可选，不填则为验证时的，可能会导致漏洞)
- properties (可选，皮肤等信息，后续可能会二次封装)
# 需要的事件
## YggdrasilAuth事件
对OnlineAuthEvent进行二次封装，因为OnlineAuthEvent的流程控制和我们现有需求不一样  
事件抛出参数：
- userName 玩家名
- userUUID 玩家UUID
- serverId 服务器在进行握手后产生的服务器ID
- playerIp 玩家IP，来自上层，可能不存在
